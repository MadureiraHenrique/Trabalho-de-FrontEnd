Construindo Sistemas que Entendem a Mente Humana

Introdução

A Engenharia Cognitiva e de Software é uma área interdisciplinar que une os princípios da ciência cognitiva com a prática da engenharia de software. O seu principal objetivo é projetar e desenvolver sistemas computacionais que sejam não apenas funcionalmente corretos, mas também intuitivos, eficientes e fáceis de usar.

Enquanto a engenharia de software tradicional foca no ciclo de vida do desenvolvimento (requisitos, projeto, implementação, teste), a engenharia cognitiva adiciona uma camada de análise centrada no usuário, considerando suas capacidades, limitações e processos mentais, como atenção, memória e resolução de problemas.

Modelo GOMS (Goals, Operators, Methods, and Selection Rules)

O GOMS é um modelo de análise de tarefas que permite prever o tempo que um usuário experiente levará para realizar uma tarefa em uma interface. Ele decompõe a interação em quatro componentes:

  • Goals (Objetivos): O que o usuário quer alcançar.
  • Operators (Operadores): Ações básicas, físicas ou mentais (ex: clicar, pressionar tecla).
  • Methods (Métodos): Sequência de operadores para atingir um objetivo.
  • Selection Rules (Regras de Seleção): Regras que determinam qual método usar quando há mais de uma opção.

Exemplo: Copiar e Colar Texto

Objetivo: Copiar um texto de um local e colar em outro.

  Método 1 (Usando o Teclado):
      1. Operador: Mover o cursor para o início do texto.
      2. Operador: Pressionar e segurar a tecla 'Shift'.
      3. Operador: Mover o cursor para o final do texto para selecionar.
      4. Operador: Liberar a tecla 'Shift'.
      5. Operador: Pressionar a combinação de teclas 'Ctrl + C' (Copiar).
      6. Operador: Mover o cursor para o local de destino.
      7. Operador: Pressionar a combinação de teclas 'Ctrl + V' (Colar).

  Método 2 (Usando o Mouse):
      1. Operador: Mover o ponteiro do mouse para o início do texto.
      2. Operador: Pressionar e segurar o botão esquerdo do mouse.
      3. Operador: Arrastar o ponteiro até o final do texto.
      4. Operador: Liberar o botão do mouse.
      5. Operador: Mover o ponteiro para a área selecionada e clicar com o botão direito.
      6. Operador: Mover o ponteiro até a opção "Copiar" e clicar.
      7. Operador: Mover o ponteiro para o local de destino e clicar com o botão direito.
      8. Operador: Mover o ponteiro até a opção "Colar" e clicar.

Regra de Seleção:
  SE o usuário está com as mãos no teclado, ENTÃO use o Método 1.
  SENÃO, use o Método 2.

Diagramas UML (Unified Modeling Language)

A UML é uma linguagem de modelagem padrão para visualizar, especificar, construir e documentar os artefatos de um sistema de software. Como não é possível renderizar imagens complexas em HTML puro, apresentamos abaixo representações textuais de diagramas comuns.

Mapa de Navegação

Um mapa de navegação (ou fluxograma de site) é uma representação visual da arquitetura de um site ou aplicativo. Ele mostra como as páginas estão interligadas, ajudando a planejar uma experiência de usuário lógica e intuitiva.

Exemplo: Mapa de um Site Institucional Simples
